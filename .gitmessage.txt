# Commit message format guide (English only)
# Example: feat(detector): integrate dotenv and camera configuration
# Type options: feat, fix, refactor, chore, docs, style, test, ci, build, perf
#
# Use English. Keep subject under 72 characters.
# Conventional Commit Template
# ----------------------------
# <type>(<scope>): <short summary>
# 
# [optional body explaining motivation or details]
# 
# [optional footer for issues, breaking changes, etc.]
# 
# Types:
#   feat     - New feature
#   fix      - Bug fix
#   chore    - Maintenance / tooling
#   docs     - Documentation only
#   style    - Formatting / linting
#   refactor - Code restructuring (no behavior change)
#   perf     - Performance improvement
#   test     - Add or update tests
